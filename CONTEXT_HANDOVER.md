# OmniRAG - Контекст проекта для ИИ-ассистента (Handover Document)

Привет, ИИ-коллега! Если ты читаешь этот файл, значит пользователь перенес тебя в новый чат для продолжения работы над проектом **OmniRAG**. Этот документ содержит весь накопленный контекст, архитектурные решения и строгие правила, чтобы ты не сломал то, что уже отлично работает.

## 🎯 Суть проекта
**OmniRAG** — это Enterprise-grade система локального поиска и ответов по документам (Retrieval-Augmented Generation). Система 100% автономная (не отправляет данные в интернет), работает через **Ollama**, поддерживает **гибридный поиск** (FAISS + BM25) и отдает ответы с интерактивными кликабельными ссылками на исходный текст.

## 🛠 Технологический стек
- **Язык**: Python 3.10
- **LLM Провайдер**: Ollama (по умолчанию `qwen2.5:3b`)
- **Эмбеддинги**: HuggingFace (`sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`)
- **Векторная БД**: FAISS
- **Лексический поиск**: BM25 (пакет `rank_bm25`)
- **Фреймворк для ИИ**: LangChain / LangChain-Core / LangChain-Community
- **UI**: Streamlit
- **Инфраструктура**: Docker & Docker Compose (с поддержкой горячей замены `Hot-Reload` для папки `src/`)

## ⚠️ Критические архитектурные решения (ЧИТАТЬ ВНИМАТЕЛЬНО!)

1. **CustomEnsembleRetriever (Проблема с LangChain)**
   В последних версиях `langchain` оригинальный класс `EnsembleRetriever` переехал и ломает импорты. Поэтому в файле `src/core/rag.py` был написан собственный **`CustomEnsembleRetriever`**. **Никогда** не пытайся вернуть `from langchain.retrievers import EnsembleRetriever`. Используй наш кастомный класс!

2. **Защита Markdown-таблиц**
   В файле `src/indexing/indexer.py` реализована умная защита таблиц от разрыва при чанкинге. Скрипт режет длинные таблицы каждые 5 строк, но *обязательно дублирует заголовок таблицы в каждом чанке*, чтобы эмбеддинг не терял смысл. Не сломай этот regex-алгоритм!

3. **Горизонтальные источники (UI)**
   В `src/frontend/app.py` кнопки-цитаты (источники) выводятся **горизонтально** через `st.columns(4)`. Пользователь явно просил сделать их компактными, не переделывай их в вертикальный список.

4. **Проверка на галлюцинации (Отключена)**
   Пользователь явно попросил **убрать** цепочку двойной верификации (поиск галлюцинаций), так как она сильно замедляла работу. Не пытайся внедрить ее обратно без явного приказа.

5. **Горячая замена кода (Hot-Reload)**
   Папка `./src` проброшена в контейнер через `volumes` в `docker-compose.yml`. Это значит, что любые изменения python-файлов в `src/` применяются мгновенно без пересборки Docker.

6. **Единый Config**
   Абсолютно все пути (`FAISS_INDEX_PATH`, `DATA_DIR`), имена моделей и хосты (`OLLAMA_HOST`) импортируются **строго** из `src/core/config.py`. Никаких захардкоженных путей в других файлах!
